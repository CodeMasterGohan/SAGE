## 2024-05-18 - Keyboard accessibility on dynamic template literals
**Learning:** In Single Page Applications (SPAs) like this dashboard that dynamically generate UI lists (e.g., search suggestions, library navigation), it's very easy to miss keyboard accessibility on the generated items. Because they are often injected as HTML strings via `.innerHTML = ...map(...)`, standard HTML attributes for accessibility (`role="button"`, `tabindex="0"`) and critical keyboard event handlers (`onkeydown`) to mimic `onclick` must be explicitly added to the string templates so screen readers and keyboard users can navigate them properly.
**Action:** When creating or reviewing dynamically rendered interactive lists, always check that the template literal strings explicitly define `role`, `tabindex`, keyboard event handlers (for Enter/Space), and focus-visible classes (`focus-visible:ring-2`) to ensure full accessibility parity with static HTML.
